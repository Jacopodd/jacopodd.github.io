AInkStudio

Applicazione desktop per simulare tatuaggi su un modello 3D del corpo umano: carichi una foto, il tatuaggio viene isolato tramite API di tipo Gemini e proiettato sul corpo in modo realistico. Ideale per privati, tatuatori e studi che vogliono mostrare anteprime credibili prima di incidere la pelle.

Problema e contesto d'uso

AInkStudio nasce per rispondere a un'esigenza semplice: vedere come starà un tatuaggio sul proprio corpo prima di farlo davvero. L'obiettivo è offrire una simulazione 3D credibile, gestita da un flusso snello di caricamento, pulizia e posizionamento del tatuaggio. :contentReference[oaicite:1]{index=1}

Utenti principali

  • Privati che vogliono provare tatuaggi su di sé prima di decidersi.
  • Tatuatori / studi che vogliono mostrare anteprime realistiche ai clienti.
  • Artisti grafici interessati ad adattare le proprie grafiche alla superficie del corpo umano. :contentReference[oaicite:2]{index=2}

Obiettivi principali

  • Simulare tatuaggi 2D su un modello 3D personalizzato del corpo.
  • Isolare automaticamente il tatuaggio dallo sfondo tramite Gemini API.
  • Organizzare tatuaggi per parte del corpo (braccia, gambe, schiena...).
  • Offrire un flusso di lavoro fluido per studi e professionisti.

Configurazione modello 3D

Al primo avvio l'utente seleziona:

  • Sesso: uomo / donna
  • Tipo di corpo: magro / normale / robusto

In base alla combinazione scelta viene caricata la mesh 3D corrispondente, presente negli asset dell'applicazione (una mesh distinta per ogni combinazione). :contentReference[oaicite:3]{index=3}

Guida utente

Flusso desktop: dalla foto al tatuaggio 3D

Qui sotto trovi il flusso pensato per la prima versione desktop (Windows/macOS), organizzato in step. È la base per l'implementazione dell'MVP. :contentReference[oaicite:4]{index=4}

1. Accesso e configurazione iniziale

  • L'utente effettua l'accesso tramite Firebase Authentication.
  • Al primo login seleziona sesso e tipo di corpo.
  • Il modello 3D corrispondente viene caricato e mostrato nella scena principale.

2. Libreria tatuaggi

La libreria è organizzata in:

  • cartella principale "Caricamenti";
  • cartelle per parte del corpo, ad esempio: Braccio sinistro, Braccio destro, Gamba sinistra, Gamba destra, Torace, Schiena, Collo, con possibilità di estenderle. :contentReference[oaicite:5]{index=5}

3. Caricamento di un tatuaggio da file

  • Dalla cartella "Caricamenti", l'utente clicca su "Carica immagine" e seleziona un file (JPEG, PNG, ecc.).
  • L'immagine viene inviata alle Gemini API, che restituiscono il tatuaggio isolato con sfondo trasparente (es. PNG).
  • Il risultato viene salvato in Firebase Storage e registrato in Cloud Firestore.
  • Il nuovo tatuaggio compare nella cartella "Caricamenti", pronto per essere trascinato sul modello. :contentReference[oaicite:6]{index=6}

4. Applicazione sul modello 3D

  • L'utente trascina un tatuaggio dalla libreria sulla superficie del modello 3D.
  • Il sistema proietta il tatuaggio sulla mesh (tecnica tipo decal) e lo adatta alle curve del corpo.
  • L'utente può:
    • spostare il tatuaggio sulla superficie;
    • ruotarlo;
    • ridimensionarlo;
    • modificare opacità e altri parametri estetici.
  • Alla conferma:
    • vengono salvate posizione, rotazione, scala, opacità;
    • il sistema identifica la parte del corpo (es. braccio sinistro);
    • il tatuaggio viene spostato automaticamente dalla cartella "Caricamenti" alla cartella corrispondente alla parte del corpo. :contentReference[oaicite:7]{index=7}

5. Modifica e rimozione

  • I tatuaggi già applicati possono essere riposizionati, ruotati o ridimensionati.
  • In caso di rimozione, l'utente può scegliere se:
    • lasciarlo nella cartella della parte del corpo;
    • riportarlo in "Caricamenti";
    • eliminarlo definitivamente dal proprio account. :contentReference[oaicite:8]{index=8}
Roadmap

Estensione mobile (Android / iOS)

Dopo la prima versione desktop è prevista un'app mobile per velocizzare il caricamento dei tatuaggi e integrare fotocamera e sincronizzazione automatica. :contentReference[oaicite:9]{index=9}

Funzionalità previste

  • Scatto diretto della foto del tatuaggio tramite fotocamera.
  • Invio della foto alle Gemini API per l'estrazione del tatuaggio.
  • Salvataggio automatico su Firebase (Storage + Firestore).
  • Sincronizzazione con lo stesso account dell'app desktop.

Workflow utente

  • L'utente apre l'app mobile e scatta una foto del tatuaggio.
  • La foto viene elaborata dalle API Gemini.
  • Il tatuaggio isolato viene salvato nella cartella "Caricamenti".
  • Quando l'utente apre l'app desktop, trova il tatuaggio già disponibile nella libreria, pronto per essere applicato sul modello. :contentReference[oaicite:10]{index=10}
Architettura

Scelte tecniche e stack

L'architettura è pensata per sfruttare tecnologie web moderne in un contesto desktop, con un backend cloud leggero basato su Firebase e integrazione con Gemini API. :contentReference[oaicite:11]{index=11}

Componenti principali

  • Client desktop:
    • Electron o Tauri per il wrapper desktop.
    • UI in HTML / CSS / JavaScript (opzionale React).
    • Motore 3D basato su Three.js (o Babylon.js). :contentReference[oaicite:12]{index=12}
  • Servizi cloud:
    • Firebase Authentication per login/registrazione.
    • Cloud Firestore come database principale.
    • Firebase Storage per le immagini dei tatuaggi.
    • Gemini API per l'estrazione e pulizia del tatuaggio. :contentReference[oaicite:13]{index=13}
  • Backend opzionale (Node.js):
    • Nascondere le chiavi delle API.
    • Gestire logica di abbonamenti/licenze.
    • Eseguire eventuale post-processing immagini. :contentReference[oaicite:14]{index=14}

Schema dati Firebase (semplificato)

In Firestore sono previste almeno tre collezioni principali:

  • users: dati base utente (nome, email, subscriptionType, createdAt, lastLogin) + configurazione del corpo (gender, bodyType).
  • tattoos: riferimento al proprietario (ownerUid), nome, parte del corpo (bodyPart), imageUrl, timestamps.
  • placements: posizione 3D del tatuaggio (position, rotation, scale, opacity), referenziando tattooId e ownerUid, con eventuale projectId. :contentReference[oaicite:15]{index=15}
